# `server/db`

Database access layer.

- `prisma.ts` — the singleton Prisma Client (server-only). Import `prisma` from `@/server/db/prisma`
  in server-only code (Server Components, Route Handlers, Server Actions, services). Never import
  this from a client component.
- `generated/` — the Prisma Client generated by `pnpm db:generate`. **Not committed to git** and
  never hand-edited; regenerate it any time you change `prisma/schema.prisma`.

The Prisma schema itself lives at `prisma/schema.prisma` (Prisma's conventional location), with
migrations in `prisma/migrations/`. See the root `README.md` → "Local Database" section for the
full Docker/Prisma workflow.

Only a minimal `SystemHealth` connectivity model exists so far — the full domain schema (Vehicle,
User, Enquiry, SyncEvent, etc.) is deferred to the dedicated database-design phase.
